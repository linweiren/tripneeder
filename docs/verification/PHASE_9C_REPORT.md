## 1. 實作項目清單

### [9C-1 & 9C-2] Prompt 交通與距離規則優化
- **實作內容**：於 `src/services/ai/tripPlanPrompt.ts` 加入硬性規則（第一站 ≤ 2km）與軟性建議（單趟 ≤ 30分、總交通 ≤ 25%）。
- **驗收標準**：AI 產出的 JSON 需包含符合邏輯的交通時長與地點安排。
- **依賴說明**：**需等 9D 完成**。目前 AI 是以其「內建知識」預估座標與距離，可能存在幻覺。須透過 9D 的 Google Places 驗證後，才能以真實經緯度計算並確認 AI 是否確實遵守此規則。

### [9C-3] 定位後反查地名 (Reverse Geocoding)
- **實作內容**：
    - 新增 `api/geocode.ts` Vercel Function 作為後端代理。
    - `HomePage.tsx` 定位成功後呼叫代理 API，將座標轉換為「縣市+行政區」（如：高雄市鼓山區）。
- **驗收標準**：
    - [ ] 點擊「使用目前位置」後，UI 顯示「座標抓取成功，正在反查地名...」。
    - [ ] 成功後，UI 顯示「目前位置：[地名]」，且「輸入地區或地點」欄位自動填入該地名。
- **當前狀態**：**可立即驗收**（需確保環境變數已設定）。

### [9C-4] 起點中心規劃強化
- **實作內容**：在 Prompt 中明確要求 AI 以使用者起點為中心向外擴張，避免跨區過遠。
- **依賴說明**：**需等 9D 完成**。需透過 9D 取得景點真實地址與座標，才能量化驗證「擴張範圍」是否符合預期。
